# Product Principles

## Purpose

Cushion helps people make confident everyday financial decisions.

The product reduces uncertainty by showing whether today's decisions will create problems before the next salary.

## Core principle

> **Every feature should help the user's future self.**

A feature belongs in the product only if it improves a future financial decision or reduces future financial stress.

## Principles

### 1. Planning over accounting

Cushion is a planner, not an accounting app.

The product focuses on what will happen before the next salary. Historical data is useful only when it improves future planning.

### 2. Clarity over completeness

The app should collect and show only the information needed for planning.

Missing non-essential detail is preferable to additional cognitive load.

### 3. Simplicity over flexibility

Prefer the simplest flow that solves the main user task.

Do not add settings, fields or alternative flows unless the MVP cannot work without them.

### 4. The system should reduce maintenance

Reuse previous information, suggest sensible defaults and minimise repetitive actions.

The user should not have to maintain a complex financial system.

### 5. One primary purpose per screen

Each screen should have one clear main task.

Secondary information and actions must not compete with it.

## MVP non-goals

The MVP does not:

- provide detailed financial analytics;
- replace accounting software;
- support investments;
- support savings goals;
- support shared or family budgets;
- sync across devices;
- require an account;
- encourage tracking for the sake of tracking.
