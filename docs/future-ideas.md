# Cushion — идеи после MVP

Этот файл хранит согласованные направления для будущих версий. Они не входят в текущий MVP и не должны реализовываться без отдельного решения.

## Автоматическое ранжирование категорий по сумме расходов

Цель: показывать категории от самых крупных расходов к самым мелким, когда в приложении накопится достаточно данных.

Предварительная логика:

1. Для нового пользователя используется зафиксированный исходный порядок категорий.
2. Автоматическое ранжирование начинается только после появления завершённых периодов.
3. Для каждой категории считается средняя фактически потраченная сумма за последние три завершённых периода. Если завершённых периодов меньше трёх, используются все доступные.
4. Расходы одной категории из разных типов суммируются.
5. Для повседневных расходов учитывается фактически потраченная сумма, а не установленный лимит.
6. Обязательный или разовый расход учитывается один раз независимо от статуса `предстоит` или `оплачено`. Смена статуса не добавляет расход повторно.
7. Категории сортируются по средней сумме от большей к меньшей.
8. Категории без расходов располагаются после остальных и сохраняют исходную взаимную последовательность.
9. Порядок обновляется один раз при создании нового периода и не меняется внутри текущего периода.
10. Если пользователь вручную переставил категории, его порядок получает приоритет и автоматическая сортировка больше его не перезаписывает.

Перед реализацией нужно проверить на реальных данных, действительно ли среднее за три периода создаёт полезный и достаточно стабильный порядок.
