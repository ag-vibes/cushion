# Cushion — дизайн-система (визуальный слой)

> Это описание **только визуального слоя**: цвета, типографика, отступы, компоненты.
> Продуктовая логика, тексты интерфейса, формулы расчётов и терминология зафиксированы в
> `product-spec.md` и `product-principles.md` — при обновлении интерфейса их менять нельзя.
> Ничего из них (статусы `предстоит`/`оплачено`, формулу свободных денег, состав экранов,
> русскую лексику в нижнем регистре, английские lowercase-категории и т.д.) не трогать.

## 1. Принцип

Тёплое, уютное пространство. Помогает человеку позаботиться о своём будущем.
Один основной акцент, спокойный нейтральный фон, никаких резких контрастов и лишнего декора.

Правило: не более 1–2 акцентных цветов на экран одновременно (primary + при необходимости danger).

### Прогресс-бар категории (повседневные расходы)

Трек — `--color-surface` (ceramic). Заливка зависит от % потраченного от лимита:

- `< 50%` → `--color-primary` (`#8FBB7C`)
- `50–90%` → `--color-accent` (`#F3A732`)
- `≥ 90%` (включая превышение) → `--color-danger` (`#ED583F`)

Остаток (`лимит − потрачено`) выводится текстом того же цвета danger, если он отрицательный, иначе `--color-ink`.

## 3. Типографика

Шрифты — Google Fonts:

- **Nunito** (600/700/800/900) — заголовки, крупные суммы, логотип/вордмарк.
- **Manrope** (400/500/600/700/800) — тело, заголовки секций, кнопки, подписи.

| Стиль                                                  | Шрифт   | Насыщенность       | Размер                      |
| ------------------------------------------------------ | ------- | ------------------ | --------------------------- |
| Крупная сумма (hero)                                   | Nunito  | 800                | 46px                        |
| Заголовок экрана / логотип                             | Nunito  | 800                | 19–28px                     |
| Заголовок секции/карточки                              | Manrope | 600                | 16px                        |
| Название категории (ниже заголовка секции по иерархии) | Manrope | 500                | 14.5px, цвет `#5C534B`      |
| Тело / метаданные («потрачено … · запланировано …»)    | Manrope | 400                | 12.5–13.5px, цвет `#9C9188` |
| Кнопка                                                 | Manrope | 700                | 16px                        |
| Таб навигации                                          | Manrope | 500 (700 активный) | 15px                        |

### Единая шкала компонентов

- заголовок группы над карточкой — Nunito 800, 22px;
- заголовок внутри карточки — Manrope 600, 16px;
- любая строка настройки или категории — Manrope 400/500, 14.5px;
- значение или сумма справа в строке — Manrope 400, 14.5px;
- подпись под строкой — Manrope 400, 12.5px;
- заголовок модального окна — Nunito 800, 21px;
- нижняя навигация — Manrope 500/700, 15px;
- чекбокс во всех формах — 18×18px; подпись — Manrope 400, 14.5px.

Один и тот же тип элемента на разных экранах использует один компонент и один размер. Строки на экранах «период» и «ещё» не должны отличаться по типографике.

Числа — простым крупным текстом, без градиентов, колец и лишнего декора.
Формат чисел и вся русская лексика (нижний регистр и т.д.) — по `product-spec.md`, раздел 3 и 5.

## 4. Форма и поверхности

- Радиус карточек: 24px.
- Радиус hero-карточки периода: 28px.
- Радиус кнопок: 18px.
- Границы карточек: `1px solid rgba(43,36,32,0.06)` (почти невидимая, тепло-нейтральная).
- Разделитель строк списка: `1px solid rgba(43,36,32,0.07)`.
- Тени — мягкие, только под primary cta-кнопкой: `0 8px 20px rgba(143,187,124,0.4)`.

## 5. Маскот — «cushion»

Минималистичная плоская иллюстрация кремово-белой подушки. Один персонаж, одно спокойное дружелюбное выражение лица (задел на настроения — в будущих версиях). Общий характер — очень простой и аккуратный doodle, консистентный во всех размерах.

Конструкция:

- Тело: кремово-белая подушка со слегка волнистым силуэтом и мягко выступающими уголками, как у настоящей подушки.
- Контур: тонкий тёмный `#2B2420`, одинаковой толщины по всему силуэту.
- Объёмные градиенты и внутренние тени не используются.
- В углах — небольшие складки: 2–4 коротких штриха, выполненных тем же контуром.
- Лицо: два круглых глаза и маленькая улыбка, цвет `#2B2420`.

Используется в трёх вариантах:

1. Основной маскот (~90px).
2. Иконка приложения: скруглённый квадрат с фоном primary или yolk, внутри — белая подушка с лицом.
3. Мини-версия для логотипа/шапки (30–36px).

### Принципы иллюстраций

- Flat design: плоские цветовые заливки без объёмных градиентов.
- Реалистичные тени и внутренние тени не используются.
- Толщина контура одинакова во всех элементах одной иллюстрации.
- Минимум деталей: только элементы, необходимые для узнаваемости и выражения характера.
- Ощущение мягкости создаётся формой и слегка волнистым силуэтом, а не градиентами или светотенью.

## 6. Компоненты экрана «главная»

Порядок и содержимое блоков — из `product-spec.md`, раздел 11.1. Ниже — только визуальная спецификация.

**Шапка**: маскот (мал.) + вордмарк «cushion» (Nunito 800).

**Hero-карточка периода**: фон — мягкий градиент `#F1F3E6 → #C9E0B9` (нейтрально-кремовый в primary-зелёный, без тёплого ceramic-подтона — чтобы не «грязнить» зелёный). Внутри: даты периода, дни до зарплаты (мелкий текст `#6E7C63`), крупная сумма свободных денег (Nunito 800, 46px, цвет `#4B7A3E`), подпись «свободные деньги».

**Карточки-секции** (обязательные платежи / повседневные расходы / разовые расходы / импульсивные покупки): белая поверхность, radius 24px, паттерн заголовок + либо пустое состояние (текст `#9C9188`), либо список строк.

**Строка категории** (повседневные расходы): название категории слева сверху (ниже по иерархии, см. §3), под ним метаданные «потрачено X ₽ · запланировано Y ₽»; остаток — справа, крупнее и жирнее; под всем — тонкий прогресс-бар (5px, radius 3px) с цветом по правилу из §2.

**CTA «добавить расход»**: sticky-кнопка над нижней навигацией, primary-фон, тёмный текст, radius 18px.

**Нижняя навигация**: 3 пункта, равные колонки (`grid-template-columns: repeat(3, 1fr)`, не `space-around` — иначе средний пункт визуально не по центру), активный — primary-текст + вес 700, неактивный — `#9C9188` + вес 500.

## 7. Не менять при переносе

- любые тексты интерфейса, термины и регистр — берутся из `product-spec.md`;
- любую бизнес-логику, формулы, статусы, флоу создания периода — не трогать;
- английские lowercase-категории расходов — не переводить и не менять регистр;
- структуру экранов и порядок блоков — сохранять как в спеке.
